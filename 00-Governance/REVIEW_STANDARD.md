# Review Standard

## Purpose

This document defines the process and criteria for reviewing specification documents, architecture decisions, and implementation code in the Aurora Studio project.

## Scope

- Specification reviews (documents in `aurora-spec`)
- Architecture reviews (ADRs, module specs)
- Implementation reviews (code in `aurora-studio`)
- AI agent output reviews

## Non-Goals

- This document does not define how to write specifications.
- This document does not define AI agent behaviour.
- This document is not a style guide.

## Ownership

Owner: Aurora Labs
Change authority: Architecture review required.

## Change Rules

- Changes require an approved RFC.
- Review criteria may not be loosened without an ADR.
